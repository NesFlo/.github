# Documentation Philosophy

Our documentation follows a few simple principles:

- Documentation is part of the product.
- Every major engineering decision should be documented.
- Documentation should evolve alongside the code.
- Clarity is preferred over complexity.
- Examples should accompany technical concepts whenever practical.
